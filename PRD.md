# 회의 자동 정리 웹 애플리케이션 PRD

## 1. 앱 개요 및 목표
- AI 보조 없이도 참여자별로 회의 발언을 구조화하고, 실행 항목(Action Item)과 기존 자료 업데이트안을 자동으로 생성하는 단일 페이지 웹 앱을 제공한다.
- 사용자는 회의록 스크립트와 기존 회의 자료 텍스트를 입력하면, 버튼 클릭 한 번으로 세 가지 결과물을 얻을 수 있다.

## 2. 핵심 기능 명세
### 2.1 참여자별 발언 요약
- 입력 스크립트를 줄 단위로 분리하고 `참여자: 발언` 패턴을 파싱한다.
- 참여자별로 발언을 묶어 발언 수, 핵심 키워드 기반 주요 발언 하이라이트, 시간 순 정리 텍스트를 출력한다.
- 발언 요약은 "참여자명 - 총 발언 수" 형식의 헤더 아래에 주요 문장을 bullet 형태로 나열한다.

### 2.2 Action Item 담당자 추출
- 발언 중 행동 지시, 일정, 요청을 나타내는 키워드를 탐지한다.
- 기본 담당자는 발언자이며, 문장 내 다른 참여자의 이름이 언급되면 담당자를 그 사람으로 재지정한다.
- 기한은 "~까지", "내일까지" 등 패턴을 파싱하여 추출하고 없으면 "기한 미정"으로 표기한다.
- "무엇을"은 문장에서 담당자/기한 표현을 제외한 핵심 동작 구문을 간단히 정제하여 보여준다.

### 2.3 논의 기반 자료 업데이트
- 기존 회의 자료 텍스트를 유지하면서, 회의 스크립트에서 변경, 추가, 삭제, 결정 등의 키워드를 포함한 문장을 탐지한다.
- 탐지된 문장을 바탕으로 업데이트 제안 bullet 리스트를 생성하고 기존 자료 하단에 "업데이트 제안" 섹션으로 병합한다.
- 중복 문장은 제거하며, 기존 자료와 비교하여 새로운 정보만 반영한다.

## 3. 기술 스택
- HTML5: 레이아웃 구조와 입력/출력 요소 구성.
- CSS3: 반응형 레이아웃, 가독성 높은 타이포그래피 및 카드형 섹션 스타일링.
- 순수 JavaScript(ES6): 파싱 로직, 데이터 처리, DOM 업데이트, 입력 검증.

## 4. UI/UX 레이아웃 설계
- 헤더: 앱 이름과 간단한 설명을 상단에 배치.
- 메인 컨테이너: 좌측은 입력 패널, 우측은 결과 패널로 2열 레이아웃 구성.
  - 입력 패널: 두 개의 textarea(회의록, 기존 자료)와 안내 텍스트, 중앙 정렬된 "자동 정리 시작" 버튼.
  - 결과 패널: 세 개의 카드형 섹션으로 각기 다른 textarea를 포함하며, 상단에 제목 라벨.
- 반응형 고려: 화면 폭이 좁을 경우 컬럼이 상하로 재배치되도록 처리.
- 피드백: 버튼 클릭 시 로딩 상태를 잠깐 표시하고, 처리 후 결과 textarea에 자동 스크롤.

## 5. 데이터 처리 흐름도
1. 사용자 입력
   - 회의록 textarea에 `참여자: 발언` 형식의 텍스트 입력.
   - 기존 회의 자료 textarea에 기존 자료 텍스트 입력.
2. 파싱 & 전처리
   - 줄 단위 분리 → `:` 기준으로 참여자와 발언 추출 → 공백/불필요 문자 정리.
   - 참여자 목록, 발언 목록, 키워드 목록을 생성.
3. 로직 처리
   - 참여자별 요약 생성 로직 실행 → 발언 수 계산, 주요 문장 모음.
   - Action Item 추출 로직 실행 → 담당자, 내용, 기한 구조화.
   - 업데이트 제안 로직 실행 → 기존 자료 대비 신규/변경 포인트 정리.
4. 출력 구성
   - 세 개의 결과 textarea에 각각의 결과 문자열 삽입.
   - 입력 검증 실패 시 경고 메시지 표시.
